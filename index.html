How the Web Works

Before diving into HTML and CSS, let's understand the fundamental concepts of how the internet and web work.

1. Introduction

Internet vs Web - What's the Difference?

The Internet

Think of it as the infrastructure: A massive network of connected computers, cables, and servers that spans the globe. It's like the road system that connects cities.

  • Physical network of computers
  • Enables data transfer between devices
  • Includes email, file sharing, video calls
The Web

Think of it as the content: A collection of websites and web pages that you access through your browser. It's like the cars and trucks that use the roads.

  • Collection of websites and web pages
  • Accessed through web browsers
  • Uses HTTP/HTTPS protocols

2. What Happens When You Type a URL?

The Journey of a Web Request

When you type www.google.com and press Enter, here's what happens behind the scenes:

1
DNS Lookup

What happens: Your browser asks "What's the address for google.com?"

Analogy: Like looking up a phone number in a phonebook

Result: DNS returns the IP address (e.g., 142.250.190.78)

2
HTTP Request

What happens: Your browser sends a request to Google's server

Analogy: Like sending a letter asking for a specific document

Request: "Please send me the Google homepage"

3
Server Response

What happens: Google's server processes your request and sends back files

Analogy: Like receiving a package with all the pieces you need

Response: HTML, CSS, JavaScript, images, etc.

4
Browser Rendering

What happens: Your browser builds the webpage from the received files

Analogy: Like assembling a puzzle with all the pieces

Result: You see the complete Google homepage

3. Key Components of the Web

Essential Web Technologies

Client (Browser)

Definition: Your web browser (Chrome, Firefox, Safari)

Role: Requests and displays web pages

Examples: Chrome, Firefox, Safari, Edge

Server

Definition: A powerful computer that stores and serves website files

Role: Responds to requests and sends back web pages

Examples: Apache, Nginx, cloud servers

DNS (Domain Name System)

Definition: The internet's phonebook that converts domain names to IP addresses

Role: Translates human-readable URLs to computer addresses

IP Address

Definition: Unique numerical address for each device on the internet

Role: Identifies specific computers and servers

Example: 192.168.1.1 (local) or 142.250.190.78 (Google)

HTTP/HTTPS

Definition: Protocols that define how data is transferred between client and server

Role: HTTP (unsecured) vs HTTPS (secured with encryption)

Security: HTTPS encrypts data for privacy and security

Hosting & Domain

Domain: Human-readable address for your website

Examples: Hosting: Netlify, Vercel, AWS. Domain: google.com

HTML Basics

HTML (HyperText Markup Language) is the standard markup language for creating web pages.

HTML Tags

What are HTML Tags?

HTML tags are the building blocks of web pages. They tell the browser how to structure and display content. Tags are enclosed in angle brackets < > and usually come in pairs: opening and closing tags.

Key Points:
  • Opening Tag: <tagname> - starts the element
  • Closing Tag: </tagname> - ends the element
  • Content: Goes between opening and closing tags
  • Self-closing: Some tags like <img> don't need closing tags

Basic HTML Structure

Every HTML document needs these essential elements:

  • <!DOCTYPE html> - Declares this is an HTML5 document
  • <html> - Root element containing all content
  • <head> - Contains metadata (title, links, scripts)
  • <body> - Contains visible page content
<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>
<body>
    <h1>Heading 1</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Semantic HTML

Semantic HTML uses meaningful tags that describe the content's purpose, making your code more readable and accessible.

What is Semantic HTML?

Why Semantic HTML Matters

Semantic HTML uses tags that clearly describe their content's purpose. Instead of using generic <div> tags everywhere, semantic tags tell both browsers and developers what the content represents.

Benefits of Semantic HTML:
  • Accessibility: Screen readers understand your content better
  • SEO: Search engines can better index your content
  • Maintainability: Code is easier to read and understand
  • Future-proof: Better compatibility with new technologies

CSS Selectors

What are CSS Selectors?

CSS selectors are patterns used to select and style HTML elements. They tell the browser which elements to apply styles to. Think of them as "targeting tools" for your CSS rules.

Selector Priority (Specificity):
  1. ID Selectors (highest priority) - #id
  2. Class Selectors - .class
  3. Element Selectors (lowest priority) - element

Element Selector

Syntax: element { property: value; }

Use Case: Style all elements of a specific type

Example: p { color: blue; }

Targets: All <p> elements on the page

p { color: blue; }

This paragraph is blue

Class Selector

Syntax: .classname { property: value; }

Use Case: Style multiple elements with the same class

Example: .highlight { background: yellow; }

Targets: All elements with class="highlight"

HTML: <span class="highlight">

.highlight { background: yellow; }
This text is highlighted

ID Selector

Syntax: #idname { property: value; }

Use Case: Style a unique element (use sparingly)

Example: #main { font-size: 20px; }

Targets: Element with id="main"

HTML: <div id="main">

Note: IDs must be unique on the page

#main { font-size: 20px; }
This text is larger

Advanced Selectors

Descendant Selector
div p { margin: 10px; }

Targets <p> elements inside <div>

Child Selector
div > p { color: red; }

Targets direct <p> children of <div>

Pseudo-class

CSS Variables (Custom Properties)

What are CSS Variables?

CSS Variables (also called Custom Properties) allow you to store reusable values in your CSS. They make it easy to maintain consistent styling across your website and make changes in one place.

Benefits of CSS Variables:
  • Consistency: Use the same colors, fonts, and spacing throughout
  • Maintainability: Change values in one place, updates everywhere
  • Readability: Meaningful names instead of magic numbers
  • Dynamic: Can be changed with JavaScript

Defining CSS Variables:

Syntax: Variables are defined with --variable-name and used with var(--variable-name)

  • :root - Global scope (available everywhere)
  • --variable-name - Must start with two dashes
  • var(--variable-name) - Function to use the variable
  • var(--variable-name, fallback) - With fallback value
:root {
    --primary-color: #3498db;
    --secondary-color: #e74c3c;
    --font-family: 'Arial', sans-serif;
    --spacing: 20px;
}

.button {
    background-color: var(--primary-color);
    font-family: var(--font-family);
    padding: var(--spacing);
}

Variables in Action:

Variable Scope

Global Scope (:root)
:root { --color: blue; }

Available throughout the entire document

Local Scope (Element)
.container { --color: red; }

Only available within that element and its children

Fallback Values
color: var(--text-color, black);

Use fallback if variable is not defined

What is Responsive Design?

Why Responsive Design Matters

Responsive design ensures your website looks and works great on all devices - from mobile phones to desktop computers. It adapts the layout, content, and functionality based on screen size.

Responsive Design Principles:
  • Mobile-First: Design for mobile first, then enhance for larger screens
  • Fluid Grids: Use flexible layouts that adapt to screen size
  • Flexible Images: Images that scale without breaking layout
  • Media Queries: CSS rules that apply based on screen characteristics

Media Queries

Understanding Media Queries

Media queries allow you to apply different CSS styles based on device characteristics like screen width, height, orientation, and resolution.

Basic Media Query

/* Mobile styles (default) */
.container {
    width: 100%;
    padding: 10px;
}

/* Tablet styles */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
}

/* Desktop styles */
@media (min-width: 1024px) {
    .container {
        width: 1000px;
    }
}

Breakpoints:

  • 768px - Tablet and up
  • 1024px - Desktop and up
  • 1200px - Large desktop

Common Media Query Patterns

Mobile-First (Recommended)
@media (min-width: 768px) { /* styles */ }

Start with mobile styles, add for larger screens

Desktop-First
@media (max-width: 767px) { /* styles */ }

Start with desktop styles, override for smaller screens

Orientation
@media (orientation: landscape) { /* styles */ }

Apply styles based on device orientation

High-DPI Screens
@media (-webkit-min-device-pixel-ratio: 2) { /* styles */ }

Target retina/high-resolution displays

CSS Transitions

What are CSS Transitions?

CSS transitions allow you to smoothly change property values over a specified duration. They create smooth animations between different states of an element.

Transition Properties:
  • transition-property - Which properties to animate
  • transition-duration - How long the animation takes
  • transition-timing-function - The speed curve
  • transition-delay - When to start the animation

Basic Transition

    transition: background 0.3s ease;
}

.button:hover {
    background: red;
}

Multiple Properties

CSS Animations

What are CSS Animations?

CSS animations let you create complex animations using keyframes. Unlike transitions, animations can have multiple steps and can run automatically.

@keyframes Animation

@keyframes bounce {
    0% { transform: translateY(0); }
    50% { transform: translateY(-20px); }
    100% { transform: translateY(0); }
}

.bouncing-element {
    animation: bounce 2s infinite;
}

Loading Animation

@keyframes spin {
    from { transform: rotate(0deg); }
    to { transform: rotate(360deg); }
}

.loader {
    animation: spin 1s linear infinite;
}

Web Accessibility

Learn how to make your websites accessible to all users, including those with disabilities.

What is Web Accessibility?

Why Accessibility Matters

Web accessibility means making websites usable by people with disabilities. This includes users with visual, auditory, motor, and cognitive impairments.

Accessibility Benefits:
  • Inclusive Design: Websites work for everyone
  • Legal Compliance: Many countries require accessibility
  • Better SEO: Search engines favor accessible sites
  • Mobile Friendly: Accessible sites work better on mobile

WCAG Principles (Web Content Accessibility Guidelines)

Perceivable

Information must be presentable to users in ways they can perceive

  • Alt text for images
  • Captions for videos
  • Color is not the only way to convey information
Operable

User interface components must be operable

  • Keyboard navigation
  • Enough time to read content
  • No content that causes seizures
Understandable

Information and operation must be understandable

  • Readable and predictable
  • Input assistance
  • Clear error messages
Robust

Content must be robust enough to be interpreted by assistive technologies

  • Valid HTML
  • Proper ARIA labels
  • Compatible with screen readers

Accessibility Tools

Testing Tools

  • WAVE: Web accessibility evaluation tool
  • axe DevTools: Browser extension for testing
  • Lighthouse: Chrome DevTools accessibility audit
  • Color Contrast Analyzer: Check color contrast ratios

Screen Reader Testing

  • NVDA: Free screen reader for Windows
  • VoiceOver: Built-in screen reader for Mac
  • JAWS: Popular screen reader for Windows
  • TalkBack: Android screen reader

Keyboard Navigation

  • Tab: Navigate through interactive elements
  • Enter/Space: Activate buttons and links
  • Arrow Keys: Navigate menus and lists
  • Escape: Close modals and menus
